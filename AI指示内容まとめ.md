# 薬歴AIナビゲーション — AI指示内容まとめ

## 1. モード概要

| モード | コンテキストメニュー | 出力セクション | 書き込み動作 |
|--------|---------------------|---------------|-------------|
| SOAP | 薬歴をAIでナビゲーション（SOAP） | 【S】【O】【A】【P】 | 上書き |
| OAP | 薬歴をAIでナビゲーション（OAP） | 【O】【A】【P】 | 追記 |
| A単独 | 薬歴をAIでナビゲーション（A） | 【A】のみ | 追記 |

- SOAPモード：テキストエリアの内容を全置換する
- OAP/A単独モード：テキストエリアの既存テキストの末尾に改行2つ挟んで追記する

## 2. 処方タイプ別の書き分けルール

AIはまず処方内容を分析し、処方タイプを判断してから書き分ける。

### 2-1. 内服薬が含まれる場合

#### 【S】（SOAPモードのみ）
- 初回投薬：その薬が必要になった症状・訴えを記載（「特に変わりない」は禁止）
- 追加処方：新しい薬が必要になった症状・訴えを記載（「体調変わりない」は禁止）
- 定期処方のみ：患者の訴えを記載（体調良好、特になし等）
- 前回の【P】の内容についても問題なかった旨を記載

#### 【O】（SOAP/OAPモード）
- 客観的所見を記載
- 副作用確認（「〜→なし」）は定期処方の薬のみ（ランダムに2種類選択）
- 初回投薬・追加処方の薬はOで副作用確認しない（まだ服用開始前のため）
- 薬品名ごとに改行して見やすく記載
- 過去2回との医薬品一致率50%以上で「定期処方」と記載
- 過去からの変更点（追加・削除・用量変更）を具体的に記載

#### 【A】（全モード共通）
- 服薬状況良好と判断した旨を記載
- 定期処方薬：ランダムに1種類選び処方妥当性を記載
- 初回投薬・追加処方：想定される副作用の注意喚起（Oではなく、Aで行う）
- 生活上・服薬上の問いかけを疑問文で記載
- 「次回までに〜」は使わない

#### 【P】（SOAP/OAPモード）
- 次回経過観察とする
- 定期処方薬：Aの問いに対し「今回はみられないが引き続き経過観察」形式
- 初回投薬・追加処方・用量変更の薬：Aで注意喚起した副作用を次回確認事項として記載
- ランダムに1種類選び次回確認すべき副作用を記載
- 「問題なかった」「問題ない」「問題なさそう」は禁止
- ただし「みられない」が不自然な場合（理解・遵守・管理など非症状系の問い）は意味が通る表現に言い換え
  - ✅「使用方法の理解に関して懸念は今回みられない」
  - ✅「食事量の低下は今回みられない」
  - ❌「使用方法は今回みられない」← 意味不明なので禁止

### 2-2. 外用薬のみの場合

| セクション | 内容 |
|-----------|------|
| 【S】 | 患者の訴え、使用部位の状態 |
| 【O】 | 使用部位の状態、使用状況 |
| 【A】 | 塗布して様子見、複数あれば併用問題なし |
| 【P】 | 次回経過観察、使用部位の状態確認 |

### 2-3. 便秘薬の場合
（テレミンソフト坐薬、センノシド錠、ピコスルファートナトリウム等、「便秘時」用法）

| セクション | 内容 |
|-----------|------|
| 【S】 | 便通に問題がある旨（「体調変わりない」「便通問題ない」は禁止） |
| 【O】 | 便通の状況、過去に他の便秘薬あれば効果不十分の旨 |
| 【A】 | 使用して様子見 |
| 【P】 | 改善傾向か悪化していないか確認（「便通は問題なかった」は禁止） |

### 2-4. 湿布・貼付薬の場合
（アドフィードパップ、ロキソプロフェンテープ、モーラステープ等）

| セクション | 内容 |
|-----------|------|
| 【S】 | 貼付部位の痛み、使用部位（胸部、腰部、両膝等） |
| 【O】 | 疼痛の状況、使用部位を明記 |
| 【A】 | 貼付して様子見 |
| 【P】 | 疼痛の改善状況を確認 |

### 2-5. 内服薬+便秘薬の併用ルール
内服薬の副作用として便秘が考えられる薬（抗コリン薬、オピオイド、Ca拮抗薬、鉄剤等）と便秘薬が同時処方の場合：

- S：便秘の症状がある旨を記載
- A：該当内服薬の副作用として便秘の注意喚起 + 水分摂取・服薬継続の文言
- P：便秘が悪化していないか、改善傾向にあるか確認

### 2-6. 酢酸亜鉛（ノベルジン等）
乳幼児かどうかで書き分けが必要。

| 対象 | S | O | A | P |
|------|---|---|---|---|
| 乳幼児 | 成長の遅れ・皮膚症状・貧血（味覚障害は禁止） | 血清亜鉛値、補充療法中 | 成長障害・皮膚炎・貧血の観点で問いかけ | 「今回はみられないが経過観察」+ 次回亜鉛値確認 |
| 小児・成人 | 味覚障害の記載可 | 同上 | 味覚障害含む観点で評価可 | 同上 |

重要：乳幼児は味覚異常を訴えられないため「味覚障害」「味覚異常」を処方理由として記載しない。

## 3. フラグ別ルール一覧

### フラグ検出方法
`#this-time` 内の `innerText` + `.mark-div span` + `.header-info span` のテキストを合わせて正規表現で判定。

| フラグ | 正規表現 | 意味 |
|--------|---------|------|
| 支B | `/支\s*[BＢb]/` | 一包化 |
| 薬B | `/薬\s*[BＢb]/` | お薬手帳未持参 |
| 薬3A | `/薬\s*[3３]\s*[AＡa]/` | 施設管理・情報交換あり |
| 薬3B | `/薬\s*[3３]\s*[BＢb]/` | 施設管理・お薬手帳情報共有 |
| 薬C | `/薬\s*[CＣc]/` | 3ヶ月以上来局なし |
| 服Bハ | `/服\s*[BＢb]\s*[ハはﾊ]/` | 服薬情報提供2（ケアマネ情報提供） |

### 各フラグの記載ルール

#### 支B（一包化）
| モード | 記載内容 |
|--------|---------|
| SOAP | SOAPのどこかに「一包化：心身の特性により適正な服用量を適正な服用時間に服用できないため一包化を行う」 |
| OAP | 同上（OAPのどこか） |
| A単独 | Aに「一包化により服薬管理を支援している。」 |
| OFF時 | 一包化に関する記述は一切行わない |

#### 薬B（お薬手帳未持参）
| モード | 記載内容 |
|--------|---------|
| SOAP/OAP | O：「お薬手帳の持参なし。」/ P：「お薬手帳を持参することで、他院や他薬局との薬の重複・相互作用の確認ができるため、次回は持参するよう説明した。」 |
| A単独 | 対象外（O/P向けルールのため） |
| OFF時 | お薬手帳未持参に関する記述は行わない |

#### 薬3A（施設管理・情報交換あり）
| モード | 記載内容 |
|--------|---------|
| SOAP/OAP | O：「施設入所中。施設職員と情報交換を行った。」/ P：「引き続き施設職員と連携し、服薬状況や体調変化について情報共有を行う。」 |
| A単独 | 対象外 |
| OFF時 | 薬3Aに関する記述は行わない |

#### 薬3B（施設管理・お薬手帳情報共有）
| モード | 記載内容 |
|--------|---------|
| SOAP/OAP | O：「施設入所中。」/ P：「施設にお薬手帳の情報を共有し、服薬管理に役立てていただく。」 |
| A単独 | 対象外 |
| OFF時 | 薬3Bに関する記述は行わない |

#### 薬C（3ヶ月以上来局なし）
| モード | 記載内容 |
|--------|---------|
| SOAP | 過去に同じ薬があっても「定期処方の継続」とは扱わず初回投薬同様に。S：現在の症状を記載（「体調変わりない」禁止）/ O：「前回来局から3ヶ月以上経過。」/ A：処方妥当性を改めて評価 / P：次回確認事項として記載（「今回はみられないが経過観察」ではなく） |
| OAP | 同上（S除く） |
| A単独 | 処方妥当性を改めて評価 |
| OFF時 | 薬Cに関する特別な扱いは不要 |

#### 服Bハ（ケアマネへの服薬情報提供）
| モード | 記載内容 |
|--------|---------|
| SOAP/OAP | O：「服薬情報提供2：担当ケアマネジャーに対し、現在の内服状況について情報提供を行った。最新の薬剤情報を提供した。」/ P：「引き続きケアマネジャーと連携し、服薬状況や処方変更があれば情報提供を行う。」 |
| A単独 | 対象外 |
| OFF時 | 服Bハに関する記述は行わない |

## 4. 禁止事項（全モード共通）

| 禁止事項 | 詳細 |
|---------|------|
| 「次回までに〜」 | Aで使用禁止。問いかけはシンプルな疑問文で |
| 「支B内服+2のため、」 | この表現は使用禁止 |
| 「変更なし。」 | 「前回処方から変更なく継続」等の別表現を使う |
| 新規作成時の「変わりない」 | 新規なので比較対象がない |
| 初回投薬・追加処方時の「特に変わりない」 | 新しい薬が出る以上、何らかの症状がある |
| マークダウン記法 | `*`、`#`、`` ` `` 等は一切禁止。プレーンテキストで出力 |
| 日本語・英単語以外の言語 | ロシア語、中国語、韓国語等は絶対に使用しない |
| 「オンライン診療」「オンライン服薬指導」「オンライン」 | 当施設ではオンライン診療を行っていないため一切使用しない |
| Pでの「問題なかった」「問題ない」「問題なさそう」 | 「今回はみられない（が経過観察）」形式を使う |
| Pでの「〜は今回みられない」が不自然な場合 | 非症状系の問い（理解・遵守・管理等）には「懸念は今回みられない」「低下は今回みられない」等、意味が通る表現に言い換える。「使用方法は今回みられない」等は禁止 |

## 5. 重視項目（ユーザー設定）

ポップアップ画面で選択可能。選択するとプロンプトに「■ 重視項目」セクションとして追加される。

### 定型項目（チェックボックス）

| 項目 | AIへの指示内容 |
|------|--------------|
| 治療 | 治療効果の評価・治療目標の達成度を重視。処方の妥当性や薬効の評価をAで重点的に記載し、Pで治療効果の確認事項を記載 |
| QOL | 患者のQOL（生活の質）を重視。日常生活への影響、睡眠・食欲・活動量などの変化をA/Pで評価・確認 |
| ADL | 患者のADL（日常生活動作）を重視。服薬管理能力、自立度、転倒リスクなどをA/Pで評価・確認 |
| 腎機能 | 腎機能への影響を重視。腎排泄型薬剤の用量調整の妥当性、eGFR/CcrなどをOに記載し、A/Pで評価・確認 |
| 副作用 | 副作用モニタリングを重視。処方薬の主な副作用を幅広くOで確認し、Aでリスク評価、Pで経過観察項目を充実 |

### フリーコメント欄
テキストエリアで自由記述可。プロンプトに「その他：{入力内容}」として追加される。

## 6. ハイリスク薬ルール

処方内容に `[H]` マークが付いている薬剤がある場合、メインセクションの下に追加セクションを出力する。

### 出力フォーマット
```
ハイリスク薬
- [薬剤名]：[注意項目のチェック内容を簡潔に記載]
（複数あれば薬剤ごとに記載）
```

### ルール
- 注意項目のチェックのみ記載（副作用詳細の確認チェックは不要）
- 患者の年齢・性別と矛盾する記載は行わない（例：高齢女性に「月経期間中の出血量に注意」は不適切）
- `[H]` マークの薬剤がない場合はセクション自体を出力しない

### 対象薬剤の例
DOAC、抗血小板薬、糖尿病用剤、精神神経用薬 など

## 7. 過去データの取り扱い

### DOM要素からの取得
| 要素ID | ラベル | 内容 |
|--------|--------|------|
| `#last-time` | 【前回の来局データ】 | 直近の来局データ |
| `#two-times-before` | 【前々回の来局データ】 | その前の来局データ |
| `#left`（フォールバック） | ラベルなし | 上記要素がない場合に一括取得 |

### フィルタリング
- 過去データから支Bの記述行（`/支\s*[BＢb]/` にマッチする行）を除外する

### AIへの指示
- 過去データは【前回の来局データ】【前々回の来局データ】のラベルで区別されている
- 「前回の処方」「前回のP」等は必ず【前回の来局データ】を参照すること
- 前回と前々回を取り違えないこと

### 定期処方の判定
- 過去2回との医薬品の一致率が50%以上であれば「定期薬」と判断

## 9. 申し送り自動抽出

SOAPモードの出力後、【P】セクションから「？」で終わる行を抽出し、`#sent-matter-textarea`（申し送り事項欄）に自動入力する。

### 対象
- 【P】または P（計画）セクション内の、末尾が「？」または「?」の行

### 動作
- 対象行が見つかった場合のみ `#sent-matter-textarea` に書き込み
- OAP/A単独モードでは【P】セクションがないため自動的にスキップされる
